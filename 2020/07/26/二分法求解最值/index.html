由1 <= D <= weights.length可知

若D==1，说明一天运载完，这时候要求的最低运载量是 sum(weights)，这是所有D取值中对应的最低运载量中的最大值，如果比这个值还大，那就不符合最低运载量这个定义了，没有必要了。

若D== weights.length，说明一天运一个即可，这时候要求的最低运载量是max(weights)，这是所有D取值中对应的最低运载量中的最小值，如果比这个值还小，那么D天内是完不成任务的 。

那么有 max(weights)<最低运载量<sum(weights)

使用用二分法，取中点mid，

如果按此运载量完不成任务，那么说明运载量太低，left = mid + 1。

如果按此运载量可以完成任务，此时运载量已经可以完成D这个要求，然而在能完成D这个要求的所有运载量中，这个运载量可能并不一定是最小的，所以右边界检修往右缩 right = mid。为何不是mid-1，因为前面强调了仅仅是可能，mid-1可能会略过答案。

410. 分割数组的最大值

给定一个非负整数数组和一个整数 m，你需要将这个数组分成 m 个非空的连续子数组。设计一个算法使得这 m 个子数组各自和的最大值最小。

注意:
数组长度 n 满足以下条件:

  • 1 ≤ n ≤ 1000
  • 1 ≤ m ≤ min(50, n)

当m==1时，res = sum(nums)

当m==n时，res = max(nums)

二分法取中点mid，即子序列的最大值是mid，按mid分组，

如果分出来的组数大于m，说明mid太小，left = mid+1

如果分出来的组数小于m，说明mid太大，right = mid-1

如果分出来的组数等于m，此时满足了要求的分组数，但是有可能并不满足此mid是能完成这个分组数的最小值，换句话说。有可能更小的mid也有可能分出来这么多组，右边界继续往左缩，为了不略过答案right = mid

875. 爱吃香蕉的珂珂

珂珂可以决定她吃香蕉的速度 K （单位：根/小时）。每个小时，她将会选择一堆香蕉，从中吃掉 K 根。如果这堆香蕉少于 K 根，她将吃掉这堆的所有香蕉，然后这一小时内不会再吃更多的香蕉。

珂珂喜欢慢慢吃，但仍然想在警卫回来前吃掉所有的香蕉。

piles.length <= H保证了只要珂珂每小时吃完一堆，那么他必可以全部吃完，即piles.length == H时，这也是他吃的速度的最大值即max(piles)

他吃香蕉的速度的最小值可以取1，之后在[1,max(piles)]区间内二分。

取mid，按mid速度吃，模拟需要用的天数

若最后时间大于H，说明速度必须加快，left = mid+1

若最后时间小于H，此时已经可以保证吃完了，但是并不能保证此时的mid是能保证吃完的的所有速度中最小的，所以右边界继续缩，和上面同样的道理，为了避免略过答案，right = mid

总结

这类题的套路都是在完成一个目标的前提下 求一个最值

首先要锁定范围

其次按mid作为猜测的答案模拟整个过程（很多小细节）

最后通过一个边界的改变完成一个这个目标，作为前提，另外一个边界的改变去求最值。
